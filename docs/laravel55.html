Laravel 5.5 Short Notes

Terminal Commands explanation
composer create-project --prefer-dist laravel/laravel blog "5.5.*" Creates project structure and installs vendor files
php artisan key:generate generates app key for the project in .env file
php artisan config:cache boosts speed
php artisan down enables maintainance mode - raises warning to user
php artisan up disables maintainance mode
php artisan storage:link links storage/app/public and public/storage directories
php artisan list make lists the available make(folder and base code scafolding) commands
php artisan make:[cmd] where cmd = [auth, command, controller, event, exception, factory, job, listener, mail, middleware, migration, model, notification, policy, provider, request, resource, rule, seeder, test]
php artisan event:generate creates scafolding for both event and listener
Routing explanation
/routes/web.php routes defined in web.php file are session based. csrf_token is required for post, put & delete
/routes/api.php routes are stateless and starts with api i.e., https://domain.tld/api/[your route here]. so no need to mention api in the routes seperately
Route::get($uri, $callback); other methods are post, put, patch, delete, option
Route::match(['get', 'post'], $callback); route is called for both get and post verbs
Route::any($uri, $callback); route is called for any http verb
Route::redirect('/here', '/there', 301); redirects route from /here to /there 301 is the redirect code
Route::view('/welcome', 'welcome', ['name' => 'Taylor']); route for just displaying view with or without direct constant.
Route::get('/home/{param}/{option_param?}', $callback); here shows how to write routing parameter and optional routing parameter
Route::get('user/{id}/{name}', $callback)->where(['id' => '[0-9]+', 'name' => '[a-z]+']); puting regex constraints on routing parameters
Route::pattern('id', '[0-9]+'); this should be put in routing service provider class to impose global contraints on id parameter
Route::get($uri, $callback)->name('profile'); this is how to name a route
$url = route('profile', ['id' => 1]); getting route url with parameter value passed
return redirect()->route('profile'); redirecting to named route
$request->route()->named('profile') return true/false to test if request has come from the named route
