Github Terminal Commands

Command explanation
git config --global user.name "Mona Lisa" configures the git user name globaly for all projects
git config user.name "Mona Lisa" configures the git user name for single project
git config --global user.name / git config user.name retrives the configured global or local variable value
git init initializes new git local repository inside new root project folder
git clone https://github.com/username/repository_name.git clones(downloads) remote repository into local drive/folder wit git origin url
git add . adds all the files and folders into git local repository
git commit -m "commit message" commits all the files and folders to compare with git local repository
git push (git push -u origin master) pushes the changes to remote git repository at origin url and master branch
git pull pulls the changes from remote git repository to local repository
git branch lists the available branch names and points at current branch name
git branch [branch] creates the branch named [branch]
git checkout [branch] / git checkout -b [branch] switches the branch. here -b is optional for creating new branch
git checkout [branch] [remote]/[branch] / git checkout --track [remote]/[branch] switches the branch (remote is required when more than one remote is available)
git merge [branch] merges the [branch] with current active branch
git branch -d [branch] deletes the branch with name [branch]
git remote -v

Outputs:
git remote set-url [origin] https://github.com/username/repository_name.git updates the git remote variable [origin] url in the local repository to new git url.
git remote add [origin] https://github.com/username/repository_name.git adds the git remote origin and its url value
git remote rename [current-name] [new-name] renames git remote variable origin from origin to origin1 etc in the local repository
git remote rm [origin] removes the git url origin and its url value
git remote show [origin]

Outputs: (typical)
Note:- switching between remotes will happen automatically when push or pull is done.
git reset --hard HEAD~ undo's the previous merge
git rebase master while you are on active branch other than master, you can resolve the conflicts in your branch before merging into master by rebase command
git rebase --continue after resolving the conflict in fist file use this command to continue for next
git rebase --skip after resolving the conflict in fist file use this command to skip for next
git rebase --abort after resolving the conflict in fist file use this command to abort rebase action
